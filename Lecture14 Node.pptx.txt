NODE.js
Node.js is a JavaScript runtime built on Chrome's V8 engine.
Allows you to run JavaScript on the server.
Non-blocking, event-driven architecture – great for web apps & APIs.

Express.js
Definition: Express is a minimal and flexible Node.js web application framework.
Purpose: Simplifies routing, handling HTTP requests, middleware.

.env
A .env file stores environment variables used to configure an application. These variables allow us to keep configuration separate from the source code.

PORT=5000
MONGO_URI=mongodb://localhost:27017/mernapp
JWT_SECRET=mysecretkey
Keeps sensitive information (API keys, passwords) out of the code
Naming convention
Environment variables are usually written in CAPITAL LETTERS with underscores.
The .env file should NOT be committed to GitHub. It is usually added to the .gitignore file to keep sensitive data private.

What is a Route?
Route decides WHICH code should run when a request comes.
Example request:
GET /api/items
The route catches the request and sends it to the correct controller.
Example:
router.get("/", getItems)
Here:"/" → the route pathgetItems → the controller function

12

Connecting Routes in index.js
const itemRoutes = require("./routes/itemRoutes");
app.use("/api/items", itemRoutes);
Meaning:
If a request starts with:
/api/items
send it to the routes file.
